## 1. Build Script (`build.rs`)

### Location
`cortex-debug-zed/build.rs`

### Functionality
The build script runs at Rust compilation time and:

✅ **Locates source files** from the sibling `cortex-debug/` project:
- `cortex-debug/dap-server.js` - DAP server entry point
- `cortex-debug/dist/` - Compiled TypeScript debug adapter

✅ **Creates bundle directory** at: `target/debug/cortex-debug-bundle/`

✅ **Copies files recursively** using the `copy_dir_all()` helper function:
- Copies `dap-server.js` to bundle
- Recursively copies entire `dist/` directory to bundle

✅ **Exports environment variable**:
- Sets `CORTEX_DEBUG_BUNDLE_DIR` for runtime access

✅ **Tracks dependencies** for incremental rebuilds:
- Monitors `dap-server.js` for changes
- Monitors `dist/` directory for changes
- Automatically re-runs on file modifications

### Key Features
- **Error handling**: Graceful warnings if source files not found
- **Logging**: Detailed build output for debugging
- **Recursive copying**: Handles nested directory structures
- **Smart rebuilding**: Only re-bundles when sources change

## Build Process Flow

```
User runs: cargo build --release
    ↓
Cargo sees build = "build.rs" in Cargo.toml
    ↓
Cargo executes build.rs
    ↓
build.rs locates source files:
├── ../cortex-debug/dap-server.js
└── ../cortex-debug/dist/
    ↓
build.rs creates bundle directory:
└── target/release/cortex-debug-bundle/
    ↓
build.rs copies files:
├── dap-server.js → bundle/
├── dist/* → bundle/dist/
    ↓
build.rs exports CORTEX_DEBUG_BUNDLE_DIR
    ↓
Cargo compiles Rust code with CORTEX_DEBUG_BUNDLE_DIR
    ↓
Extension binary ready with bundled assets
```

The Zed extension (`lib.rs`) will:
1. Receive debug configurations
2. Parse `--server=<port>` arguments
3. Spawn the bundled `node dap-server.js --server=<port>`
